# תכנית פיתוח אתר LAVERVOTREVEHICULE.FR

> אתר לחברת **AvaLior** (שם מסחרי: lavervotrevehicule) – שירות ניקוי רכבים נייד, אקולוגי, ללא מים, באזור ניס / הריביירה הצרפתית.
> קהל היעד: דוברי צרפתית (תיירים ותושבים מקומיים), בעלי קרוואנים, רכבים ואופנועים, וכן לקוחות עסקיים (סוכנויות, קמפינגים).

---

## 1. סקירה כללית של הפרויקט

האתר כולל 4 עמודים עיקריים, טופסי הזמנה לכל קטגוריית רכב, מערכת תשלומים עם חשבונית מקדמה, יומן תורים מבוסס מיקום (רדיוס של 5–10 ק"מ מניס), ומונה מים חסכוניים שמתעדכן עם כל שירות.

### צבעים מומלצים (לפי המסמך)
- **ירוק אקולוגי** – צבע ראשי (מסמל את הקיימות)
- **כחול Azur** (כחול ריביירה) – צבע משני
- **כתום / צהוב** – צבעי הדגשה ו-Call To Action

### שפות האתר
- **צרפתית (FR)** – שפה ראשית וחובה (קהל היעד צרפתי, דומיין .FR)
- **אנגלית (EN)** – מומלץ מאוד (תיירים זרים שמגיעים לריביירה)
- **אופציונלי**: איטלקית / גרמנית (תיירים מאירופה)

## 2. המלצות טכנולוגיות (Tech Stack)

### Frontend
| רכיב | טכנולוגיה מומלצת | הסבר |
|------|------------------|------|
| Framework | **Next.js 15 (App Router)** | SSR/SSG מעולה ל-SEO, ביצועים גבוהים, תמיכה מובנית ב-i18n |
| שפה | **TypeScript** | בטיחות טיפוסים, פחות באגים |
| עיצוב | **Tailwind CSS + shadcn/ui** | פיתוח מהיר, עיצוב מודרני, נגיש |
| אנימציות | **Framer Motion** | אנימציות חלקות לתחושה פרימיום |
| טפסים | **React Hook Form + Zod** | ולידציה חזקה, UX מצוין |
| תרגום | **next-intl** | תמיכה מובנית ב-Next.js ל-FR/EN |
| אייקונים | **Lucide React** | ספריית אייקונים מודרנית |

### Backend
| רכיב | טכנולוגיה מומלצת | הסבר |
|------|------------------|------|
| API | **Next.js API Routes / Server Actions** | שילוב מלא, פחות מורכבות |
| מסד נתונים | **PostgreSQL (Supabase / Neon)** | רלציוני, אמין, חינם להתחלה |
| ORM | **Prisma** | עבודה נוחה עם DB, Type-safe |
| אימות משתמשים | **NextAuth.js / Clerk** | Admin + לקוחות רשומים |
| תשלומים | **Stripe** | תומך בצרפתית, SEPA, חשבוניות אוטומטיות, עונה לרגולציית EU |
| שליחת מיילים | **Resend / SendGrid** | שליחת חשבונית ואישורי הזמנה |
| יצירת PDF | **react-pdf / pdfkit** | לחשבוניות מקדמה |
| מפות / גיאוקודינג | **Google Maps API / Mapbox** | לחישוב רדיוס 5–10 ק"מ |
| CMS לבלוג | **Sanity / Strapi / MDX** | עריכת פוסטים בלי תלות במפתח |

### DevOps / אירוח
| רכיב | טכנולוגיה מומלצת |
|------|------------------|
| Hosting | **Vercel** (פשוט ומהיר ל-Next.js) |
| Domain + DNS | **OVH / Gandi** (ספקי דומיין צרפתיים ל-.fr) |
| SSL | Let's Encrypt (כלול ב-Vercel) |
| Analytics | **Plausible / Google Analytics 4** |
| Monitoring | **Sentry** (תפיסת שגיאות) |
| Cookie Banner | **Cookiebot / CookieYes** – חובה לפי GDPR / צרפת |

## 4. מבנה מסד נתונים (סכמה בסיסית)

```
User (admin / customer)
├── id, email, name, phone, role, createdAt

Booking
├── id, userId, vehicleType (camping-car / car / moto)
├── vehicleSize, package, options[], totalPrice
├── address, lat, lng, distanceFromNice
├── scheduledDate, status (pending/confirmed/completed/cancelled)
├── paymentType (deposit / full), paymentStatus
├── stripePaymentIntentId, invoiceUrl

WaterCounter (singleton / global stats)
├── id, totalWashes, totalLitersSaved, updatedAt

BlogPost
├── id, slug, title, content, excerpt, coverImage
├── category, tags[], publishedAt, author

Availability
├── id, date, startTime, endTime, zoneLat, zoneLng, isAvailable
```

## 6. נקודות חשובות שצריך לברר מול הלקוח

1. **לוגו וזהות מותגית** – הלקוח ציין שאין עדיין לוגו ואין charte graphique. האם לייצר בינתיים או לחכות?
2. **תמונות** – מאיפה יגיעו תמונות של הרכבים, הטכנאים והשירות? (צילום מקצועי / stock)
3. **טקסטים שיווקיים** – האם הלקוח יספק טקסטים סופיים בצרפתית או שצריך copywriter?
4. **SIRET ופרטי החברה** – נדרשים לטקסטים המשפטיים ולחשבוניות
5. **חשבון Stripe** – למי יהיה הגישה? צריך פרטי חברה מאומתים
6. **Stakeholders נוספים** – האם יש אזורים נוספים מעבר ל-5–10 ק"מ מניס? (Antibes, Cannes, Monaco?)
7. **שפות** – האם להשקה ראשונית מספיקה צרפתית או גם אנגלית?
8. **בלוג** – מי יכתוב את הפוסטים? באיזו תדירות?
9. **פוליסת ביטול** – האם יש מדיניות ביטול הזמנה והחזר מקדמה?
10. **שירות B2B** – במסמך מוזכרים concessionnaires וקמפינגים – האם להקים פורטל נפרד להם או טופס יצירת קשר בלבד?
